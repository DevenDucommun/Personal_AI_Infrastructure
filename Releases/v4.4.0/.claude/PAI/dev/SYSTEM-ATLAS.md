## Core Systems

### 1. The Algorithm (PAI/Algorithm/)

The 7-phase execution engine: **Observe → Think → Plan → Build → Execute → Verify → Learn**

- **Current version:** v3.9.0 (pointed to by `LATEST` file)
- **Shipped versions:** v3.5.0, v3.7.0, v3.8.0, v3.9.0
- **Support docs:** CapabilitySelection.md, Examples.md, ISC-Methodology.md, MICRO.md
- **Concept:** Transition from CURRENT STATE to IDEAL STATE via verifiable criteria (ISC)
- **ISC = Ideal State Criteria = Verification Criteria** — binary, testable, 4-12 words each
- **Effort tiers:** Standard < Extended < Advanced < Deep < Comprehensive

### 2. Config System (config/)

Source-of-truth JSONC files compiled to settings.json by BuildSettings.ts:

## Data Flow

### Session Lifecycle

```
Session Start
  → TerminalState (init terminal)
  → LoadContext (inject relationship, learning, work context)
  → BuildCLAUDE (regenerate CLAUDE.md if needed)
  → BuildSettings (regenerate settings.json if config changed)

User Message
  → ModeClassifier (classify effort tier)
  → RatingCapture (capture ratings)
  → TerminalState (update)
  → UpdateTabTitle (set tab)
  → SessionAutoName (name session)

Tool Use
  → SecurityValidator (pre: guard Bash/Edit/Write/Read)
  → AgentExecutionGuard (pre: guard Task)
  → SkillGuard (pre: guard Skill)
  → PRDSync (post: sync PRD to work.json)

Response Complete
  → LastResponseCache (cache for ratings)
  → DocIntegrity (validate cross-refs)
  → StopOrchestrator (orchestrate stop hooks)
  → AlgorithmTracker (track phase progress)

Session End
  → WorkCompletionLearning (extract insights)
  → SessionCleanup (clear state)
  → RelationshipMemory (capture relationship notes)
  → UpdateCounts (update system counts)
  → IntegrityCheck (validate PAI integrity)
```

```
  - PAI/USER/PROJECTS/PROJECTS.md

  - Active work summary (MEMORY/WORK/)
  ↓
On-demand context (CONTEXT_ROUTING.md):
  - System docs loaded as needed
  - USER/ context loaded as needed
```
